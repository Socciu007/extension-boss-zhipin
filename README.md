# BOSS Zhipin Chrome Extension

A Chrome MV3 extension that automates recruiter workflows on [BOSS Zhipin](https://www.zhipin.com) (the largest Chinese-language professional recruitment platform). It runs **two independent loops** from a single popup:

- **Auto-reply** — replies to unread chat conversations from candidates.
- **Recommend-greet** — opens the recommended candidates tab and clicks greet on cards with a personalized opener.

Each loop has its own toggle, counter, and daily limit, and they run independently so you can enable one without disturbing the other.

## ✨ Features

### 🎯 Core

- **One-click auto-reply loop** — flip the toggle in the popup and the extension starts replying to unread conversations. The loop self-chains with human-like jitter (2-5 s) between messages and stops the moment you turn it off.
- **Context-aware replies** — each reply is generated by an LLM that sees the candidate's name, the job title they applied for, and the latest message in the conversation.
- **Three-option LLM output** — the prompt asks the model for three short reply candidates, separated by `{option1：…, option2：…, option3：…}`. The extension parses that custom format, picks one at random, and sends it.
- **Recommend-greet loop** — second independent loop that scrapes `/web/chat/recommend`, clicks *打招呼* on the first card, generates a Chinese opener from the candidate's profile, and sends it as the first message.
- **Full card scrape** — for each recommend candidate the extension captures avatar URL, online status, age, years of experience, education, job status, expected salary/location/job, highlights paragraph, tag chips, and full work-experience timeline.

### 🛡️ Safety

- **LRU replied cache** — the auto-reply loop remembers which conversations it has already replied to (up to 2,000 entries) so it never sends the same canned response twice in a row.
- **Race-guard lock** — only one loop tick runs at a time. If the service worker is killed mid-loop, the lock is released on the next SW startup.
- **Per-tab find** — the loop targets a single BOSS chat tab; if none is open, it no-ops silently.
- **URL-pattern match** — Chrome match pattern `https://*.zhipin.com/web/chat/*` (scheme explicit, host wildcard scoped, path requires a segment after `web/chat`).
- **Same-origin iframe access** — the recommend cards live inside `<iframe name="recommendFrame">` with src `/web/frame/recommend/...`. The content script reads its `contentDocument` directly (no cross-origin privileges required).

### 🖼️ Popup UI

- **280 px layout** with two control rows (Auto-reply + Recommend-greet), a 3-cell status grid (sent/limit, errors, running), and an error line with an inline *Reset today* button.
- **Polling** the service worker once per second while the popup is open.
- **Toast notifications** for every state transition (enabled, disabled, limit reached, reset).
- **Mutual-exclusion states** — when one loop is active, the other row shows *Another mode is active* in italic and its button is disabled.
- **Warning button state** when the daily limit is hit — the toggle is disabled, the label changes to "Daily limit reached", and the row turns amber.
- **Emoji rendered as HTML entities** (`&#128236;` etc.) so Tailwind v4's source scanner never crashes on 4-byte UTF-8 codepoints.

### 🔧 Developer Tools

- **[`tools/inspect-boss.cjs`](tools/inspect-boss.cjs)** — a Node script that uses Chrome DevTools Protocol to dump the live DOM of the BOSS chat page. Produces `tools/inspect-out/document.html`, `first-row.html`, `message-pane.html`, and `probe-report.json` so you can keep `dom.ts` selectors in sync as BOSS rolls new builds.
- **[`tools/dump-iframe.cjs`](tools/dump-iframe.cjs)** — a minimal CDP helper that dumps just the `recommendFrame` iframe (176 KB) for selector verification. The main `inspect-boss.cjs` cannot reach cross-frame DOM.

## 📋 Prerequisites

- **Node.js 20+**
- **Google Chrome** (or any Chromium-based browser with manifest v3 support)
- A **BOSS Zhipin** account logged in via Chrome (so the chat page can fetch your real conversations)

## 🚀 Quick Start

### 1. Install dependencies

```bash
npm install
```

### 2. Set up the LLM endpoint

The default endpoint is the internal proxy at `http://ai.dadaex.cn/backapi/chatGpt/chatAll`. CORS for that host is already whitelisted in `manifest.config.ts` via `host_permissions`.

If you want to use a different endpoint (e.g. Google Gemini directly), edit [src/background/gemini.ts](src/background/gemini.ts) and remove the `ai.dadaex.cn` host from `manifest.config.ts`.

### 3. Start the dev server

```bash
npm run dev
```

Vite will build the bundle into `dist/`.

### 4. Load the extension into Chrome

1. Open `chrome://extensions/`
2. Enable **Developer mode** (top right)
3. Click **Load unpacked** and select the `dist/` directory
4. The BOSS ZHIPIN icon should appear in your toolbar

### 5. Use the extension

1. Open `https://www.zhipin.com/web/chat/index` in Chrome (log in if needed)
2. Click the extension icon → click **Enable** under *Auto-reply* (or *Recommend-greet*)
3. The popup polls the service worker once a second. The status grid updates as replies are sent.
4. Click **Disable** to stop the loop at any time.
5. If you hit the daily limit, click **Reset today** to clear the counters.

## 🛠️ Build for production

```bash
npm run build
```

This runs `tsc -b && vite build` and produces a release artifact at `release/boss-zhipin-extension-<version>.zip` (auto-generated by `vite-plugin-zip-pack`).

## 🧪 Update selectors after a BOSS redesign

BOSS rolls new builds regularly, and class names can change. To re-verify the selectors:

1. Start a Chrome instance with the remote-debugging port:

   ```powershell
   # PowerShell
   Start-Process "C:\Program Files\Google\Chrome\Application\chrome.exe" `
     -ArgumentList "--remote-debugging-port=9222", `
     "--user-data-dir=C:\tmp\chrome-debug-profile"
   ```

2. In that Chrome, log in to BOSS.

3. **For chat-list / conversation selectors** — open `https://www.zhipin.com/web/chat/index` and click any conversation row so the right panel renders, then run:

   ```bash
   node tools/inspect-boss.cjs
   ```

4. **For recommend-card selectors** — open `https://www.zhipin.com/web/chat/recommend` (the cards live inside the `recommendFrame` iframe, so the main `inspect-boss.cjs` cannot reach them). Run:

   ```bash
   node tools/dump-iframe.cjs
   ```

   This dumps the iframe HTML to `tools/inspect-out/iframe-recommend.html`.

5. Inspect the relevant output file (or the `probe-report.json` from step 3) to see which selector candidates hit / miss.

6. Update [src/content/features/auto-reply/dom.ts](src/content/features/auto-reply/dom.ts) with the new selectors.

7. `npm run build` and reload the extension.

## 🔌 Message protocol

The service worker, content script, and popup communicate through typed message unions defined in [src/shared/messages.ts](src/shared/messages.ts).

### Popup → SW (`PopupToSw`)

| Message | Payload | Effect |
|---|---|---|
| `GET_STATE` | — | Returns current `STATE` (counters, flags, daily limit) |
| `TOGGLE_ENABLED` | `{ enabled }` | Flips the auto-reply toggle; on enable, kicks off `runOnce()` immediately |
| `TOGGLE_RECOMMEND` | `{ enabled }` | Flips the recommend-greet toggle; on enable, kicks off `runRecommendGreetOnce()` |
| `RESET_STATS` | — | Clears today's counters (sent, errors, recommend-greeted) and re-enables the loop |
| `CLEAR_REPLIED` | — | Empties the LRU replied cache |
| `UPDATE_CONFIG` | `{ config }` | Currently a no-op (API key is hardcoded) |
| `RUN_ONCE` | — | Fire-and-forget single tick (debug) |

### SW → Content (`SwToContent`)

| Message | Payload | Effect |
|---|---|---|
| `PING` | — | Health check |
| `CLICK_TAB` | `{ tab: 'chat' \| 'recommend' }` | Click the matching left-sidebar tab and wait for the view to mount |
| `SCRAPE_UNREAD` | — | Return the list of unread conversations from `/web/chat/index` |
| `SCRAPE_RECOMMENDED` | — | Return the list of recommend candidates with full profile data |
| `OPEN_CONV` | `{ convId }` | Click the row, wait for the right panel to render |
| `READ_LAST_MESSAGE` | `{ convId }` | Return the last bubble's text from the right pane |
| `GREET_CANDIDATE` | `{ cardId }` | Click *打招呼* on the matching card |
| `SEND_REPLY` | `{ convId, text }` | Type into the composer and click Send |
| `TEST_SELECTORS` | — | Run a probe for every entry in `SEL` |

## 🐛 Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `Could not establish connection. Receiving end does not exist` | Content script not injected on the BOSS tab | Reload the extension in `chrome://extensions/`, then press F5 on the BOSS tab |
| Empty string from `READ_LAST_MESSAGE` | React render raced the call, or the last bubble is a system card | Already handled with a 3 s retry + system-bubble filter in `readLastCandidateMessage` |
| `Access-Control-Allow-Origin` error on the LLM endpoint | Missing `host_permissions` for the LLM host | Add the host to `manifest.config.ts > host_permissions` |
| `toolResult` from AskUserQuestion never returns | `chrome.runtime.onMessage` listener returns `false` instead of `true` | Make sure the listener returns `true` to keep the channel open for `sendResponse` |
| Recommend cards return 0 even though the page shows them | Cards are rendered inside the `recommendFrame` iframe | Use `tools/dump-iframe.cjs` (the main `inspect-boss.cjs` cannot reach cross-frame DOM) |
| Tailwind v4 source scanner crash on certain Unicode | Tailwind v4's scanner can't handle 4-byte UTF-8 codepoints in `.tsx` | Already mitigated: emoji in `App.tsx` are stored as `&#128236;` HTML entities |
| Loop hangs after a tab switch | React unmounted the chat list but the loop kept trying to scrape | `CLICK_TAB` is fired before every scrape, so the destination view is mounted before the scrape runs |
| Popup shows *Another mode is active* in italic on one of the rows | You enabled Auto-reply while Recommend-greet was running (or vice versa) | Click the disabled row's button — it disables the other loop first, then enables itself |
