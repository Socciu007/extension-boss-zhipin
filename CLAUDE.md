# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Chrome MV3 extension **boss-zhipin-extension**. Single React + TypeScript + Vite codebase that builds into a CRX-bundle and a zip artifact for distribution to internal eptrade.cn users.

## Commands

| Task | Command |
|---|---|
| Install deps | `npm install` |
| Dev (HMR, watch rebuild) | `npm run dev` |
| Type-check + production build | `npm run build` (runs `tsc -b && vite build`) |
| Preview built bundle | `npm run preview` |
| Load into Chrome | `chrome://extensions/` → Developer mode → "Load unpacked" → select `dist/` |
| Release artifact | `release/<name>-<version>.zip` (auto-generated by `vite-plugin-zip-pack`) |

There is no test runner, linter, or formatter configured. Do not add ESLint/Prettier without confirming with the user.

## Architecture — three surfaces, one manifest

The manifest is the wiring diagram. All three UIs are wired through [manifest.config.ts](manifest.config.ts):

1. **Popup** (`src/popup/`, `action.default_popup: 'src/popup/index.html'`)
   - `main.tsx` mounts `<App />` into `#root`.
   - `App.tsx` renders a centered "BOSS ZHIPIN" title and a `#notification` div for `showToast()`.
   - `scripts.ts` exports `showToast`, `loadTab` (await tab `onUpdated` complete), `delay`, and `decodeCapcha` (calls chaojiying OCR).

2. **Side panel** (`src/sidepanel/`, `side_panel.default_path: 'src/sidepanel/index.html'`)
   - Same React boot pattern as popup. Currently a scaffolded logo demo.

3. **Content script** (`src/content/features/index.ts`, `content_scripts[0].js`)
   - Runs on **`https://*/*`** (every HTTPS page) — see Security note below.
   - Currently a stub: listens for `chrome.runtime.onMessage` of type `AUTO_LOGIN` and calls `autoLogin()` on `oocl.com`. Real automation is not implemented.

### Surfaces that are scaffolded but NOT wired to the manifest

- `src/content/main.tsx` and `src/content/views/App.tsx` — these build a React app that mounts a `<div id="crxjs-app">` into the page, but `manifest.config.ts` only points `content_scripts[0].js` at `src/content/features/index.ts`. The `main.tsx` + `views/App.tsx` chain is **not currently built or injected**. Either delete it, or update the manifest to point at `src/content/main.tsx` if the intent is to mount a React UI on the page.
- `src/utils/services.ts` — empty (1 line).
- `src/mockdata/` — empty directory.

## Key conventions

- **Path alias**: `@/*` → `src/*` (see [tsconfig.app.json](tsconfig.app.json) and `vite.config.ts` `resolve.alias`). Use `@/components/...` rather than relative paths from `src/`.
- **Tailwind v4**: configured via the `@tailwindcss/vite` plugin in `vite.config.ts`; **there is no `tailwind.config.js`** — v4 is config-free. Custom classes belong in CSS files next to the component.
- **TypeScript strictness**: `strict`, `noUnusedLocals`, `noUnusedParameters`, `noFallthroughCasesInSwitch` are on. Unused variables fail `tsc -b` (which runs before `vite build`).
- **Manifest is TS, not JSON**: edit [manifest.config.ts](manifest.config.ts) — `vite.config.ts` imports the default export and passes it to `crx({ manifest })`. The CRXJS plugin emits the real `manifest.json` into `dist/`.
- **Zip artifact**: `vite-plugin-zip-pack` runs on every `vite build`; it reads `name` and `version` from `package.json`. Bumping `version` there changes the zip filename.

## Security notes worth flagging

- **`content_scripts.matches` is `https://*/*`** but `host_permissions` is only `*://*.eptrade.cn/*`. The content script can run on every HTTPS site but can only call `chrome.*` APIs that need host permission (cookies, fetch via the page world is unrestricted). Don't add code there that reads sensitive data from non-eptrade pages.
- **Hardcoded credentials in [src/popup/scripts.ts:56-70](src/popup/scripts.ts#L56-L70)**: `decodeCapcha` posts the chaojiying account `nfwyst` / `daisikia` to `upload.chaojiying.net`. These look like test creds in a public repo; do not ship them to production — move to `chrome.storage.local` or env-injected config, and never commit real secrets.

## Where to add things

| If you want to... | Edit |
|---|---|
| Add a popup button or page | `src/popup/App.tsx` (or a new component under `src/components/`) |
| Add a new content-script feature that runs on all HTTPS pages | `src/content/features/index.ts` (or a sibling in `src/content/features/`, and reference it from `manifest.config.ts` `content_scripts[0].js`) |
| Mount a React UI inside a page | Decide first: wire `src/content/main.tsx` into the manifest, or delete it |
| Add a new Chrome permission | `manifest.config.ts` `permissions` / `host_permissions` |
| Add a new file under `src/` | Use the `@/` import alias |
| Change extension name or version shown in chrome://extensions | `package.json` `name` and `version` (manifest pulls from there) |
