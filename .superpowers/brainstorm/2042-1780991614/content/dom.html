2. Selector strategy — "stable first, fallback second"

BOSS là SPA React. Class CSS thường có hash (vd .chat-item-7f3a9c) → đổi theo build. Vì vậy:

  • Ưu tiên #1: data-attribute (vd [data-uid], [data-conv-id]) — thường ổn định.
  • Ưu tiên #2: cấu trúc DOM + vị trí (vd aside ul > li thay vì .listItem__abc123).
  • Ưu tiên #3: text content một phần (vd "chưa đọc" / "未读").
  • Ưu tiên #4 (cuối cùng): class hash, kèm comment // FRAGILE trong code.

3. Một số selector dự kiến (CẦN VERIFY khi test)

// src/content/features/auto-reply/dom.ts — tập trung vào một file
// để dễ bảo trì khi BOSS đổi markup

export const SEL = {
  // chat list (sidebar trái)
  chatListRoot:    'aside > ul, [data-component="chat-list"]',
  chatListItem:    '[data-uid], aside ul > li',   // mỗi conversation
  unreadBadge:     '[class*="badge"], [class*="unread"]',
  candidateName:   '[class*="name"]',

  // right panel: hội thoại đang mở
  messagePane:     '[class*="message-list"], [class*="chat-body"]',
  lastBubble:      '[class*="message-item"]:last-of-type, [class*="message-item"]:nth-last-child(1)',
  lastBubbleText:  '[class*="text"], [class*="content"]',

  // input + send
  input:           '[contenteditable="true"]',
  sendButton:      '[class*="send"], button[type="button"]',
}

⚠️ Selector trên là dự đoán. Khi implement, tôi sẽ mở zhipin.com/web/geek/chat thật, inspect DOM, và cập nhật cho khớp. Sẽ có 1 chế độ "selector test" trong popup để verify từng selector trả về đúng node.

4. Quy trình scrape 1 conversation

  1. findUnread(): query SEL.chatListItem trong SEL.chatListRoot → lọc các phần tử có unreadBadge descendent. Trả về { uid, name, snippet }[].
  2. openConv(uid): tìm [data-uid="<uid>"] → click() → wait.waitFor(SEL.lastBubble, { timeout: 5000 }).
  3. readLastMessage(): lấy text của lastBubble > lastBubbleText. Nếu rỗng / là "Tin nhắn hệ thống" → bỏ qua, coi như không có candidate message.
  4. Trả về string cho SW.

5. typeAndSend — gõ & gửi

// src/content/features/auto-reply/act.ts
export async function typeAndSend(text: string): Promise<void> {
  const input = await wait.waitFor(SEL.input, { timeout: 5000 })
  input.focus()

  // contenteditable không nhận value=. Cần dispatch input event.
  // Cách phổ biến:
  input.textContent = ''                        // clear
  input.append(document.createTextNode(text))   // append text
  input.dispatchEvent(new InputEvent('input', { bubbles: true }))

  // chờ React render
  await sleep(200)

  const send = await wait.waitFor(SEL.sendButton, { timeout: 3000 })
  send.click()
}

⚠️ BOSS có thể wrap input bằng component React custom. Nếu append(textNode) không làm React rerender → thử paste event với ClipboardEvent hoặc gõ từng ký tự với KeyboardEvent('keydown', { key }). Có test mode trong popup để thử 3 cách.

6. "Selector test" mode trong popup

Popup có nút Test selectors (chỉ bật khi enabled=false). Click → mở tab zhipin.com chat → chạy từng selector, highlight node match được (border đỏ), log kết quả vào popup console. Giúp user xác minh selector còn dùng được trước khi bật auto-reply thật.
