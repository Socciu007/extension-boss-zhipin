State / Storage Schema

Một store duy nhất: chrome.storage.local — không sync, không cần mã hoá cho khoá nội bộ

Các key trong chrome.storage.local

// === src/shared/types.ts ===

type AppConfig = {
  geminiApiKey: string         // user nhập trong popup, KHÔNG log
  model: 'gemini-2.0-flash'    // default; có thể đổi
  throttleMinMs: number        // default 2000
  throttleMaxMs: number        // default 5000
  dailyLimit: number           // default 200
  scheduleWindows: Array<{     // mảng cửa sổ thời gian
    days: number[]              // 0=CN..6=T7 (giống JS Date.getDay())
    start: string               // "09:00"
    end: string                 // "12:00"
  }>
  systemPrompt: string         // hardcode, không expose UI
}

type Conversation = {
  id: string                    // hash từ DOM (uid, hoặc URL fragment)
  candidateName: string         // chỉ để log
  lastSnippet: string           // 80 ký tự cuối của candidate — debug
  lastRepliedAt: number         // ms epoch; 0 nếu chưa
}

type DailyStats = {
  date: string                  // "YYYY-MM-DD" local
  sent: number
  errors: number
  lastErrorMsg: string          // rỗng nếu OK
}

type Persisted = {
  config: AppConfig
  conversations: Record<string, Conversation>   // dedupe
  stats: DailyStats
  enabled: boolean              // ON/OFF global
}

Quy tắc reset & prune

  • Daily counter: reset khi new Date().toISOString().slice(0,10) khác stats.date. Có thể thực hiện lazy ở đầu mỗi vòng runOnce().
  • Replied set: giữ tối đa 2000 ID gần nhất (LRU). Tránh storage phình vô hạn.
  • API key: chỉ đọc trong service worker. Không log. Không gửi qua message tới content script. Nếu popup cần validate → SW trả về { ok: true }, không trả key.
  • enabled=false: loop vẫn chạy (alarm vẫn fire) nhưng runOnce() return ngay. Không tốn API quota.

Migration path (nếu schema đổi)

Không cần. Schema mới, key mới, không có dữ liệu cũ để tương thích. Nếu sau này cần thêm field, dùng default qua spread { ...defaults, ...persisted } ở storage.ts.

Nếu schema ổn, tôi sang phần Schedule + throttling.
