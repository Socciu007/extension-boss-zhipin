Auto-Reply Boss Zhipin — luồng tổng quan

Một vòng lặp: tìm inbox chưa rep → scrape → Gemini → gõ & gửi → đếm quota

Sequence: 1 cuộc hội thoại được auto-reply

  ┌────────────┐         ┌─────────────────┐         ┌──────────────┐
  │ zhipin.com │         │  Service Worker │         │ Gemini API   │
  │  chat page │         │  (background)   │         │ (Google AI)  │
  └─────┬──────┘         └────────┬────────┘         └──────┬───────┘
        │                         │                          │
   1.   │  scrape chat list       │                          │
        │  (find unread conv)     │                          │
        │ ─────────────────────► │                          │
        │                         │                          │
   2.   │                         │  Gemini generate         │
        │                         │  (system prompt +        │
        │                         │   last candidate msg)    │
        │                         │ ───────────────────────► │
        │                         │                          │
   3.   │                         │ ◄───── reply text ──────│
        │                         │                          │
   4.   │  open conversation      │                          │
        │ ◄────────────────────── │                          │
        │  scrape last candidate  │                          │
        │  message (for context)  │                          │
        │ ─────────────────────► │                          │
        │                         │                          │
   5.   │  type into input        │                          │
        │ ◄────────────────────── │                          │

        │                         │  wait 2-5s               │
        │                         │                          │

Phạm vi MỘT vòng lặp

  • Bước 1-2: service worker chỉ đạo. Content script chỉ thực thi khi được yêu cầu.
  • Bước 3-4: LLM chỉ được gọi SAU khi đã scrape được nội dung — tránh gọi Gemini vô ích.
  • Bước 5-6: Gõ & gửi trên UI thật, không gọi Boss API nội bộ → dễ break khi BOSS đổi DOM, nhưng không vi phạm chính sách API.

Nếu thấy flow hợp lý, tôi sang phần Components & data flow.
