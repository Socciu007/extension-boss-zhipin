Components & Data Flow

File mới được thêm (toàn bộ) và cách chúng liên kết

1. Service Worker (background) — "bộ não"

src/background/
  index.ts          ← đăng ký chrome.alarms, message router, lifecycle
  loop.ts           ← runOnce() — orchestrate một vòng lập
  gemini.ts         ← callGenerative() — gọi Gemini REST
  scheduler.ts      ← isInActiveWindow(), nextWakeUp()
  storage.ts        ← typed wrapper quanh chrome.storage.local
  config.ts         ← constants (URL, models, throttle defaults)

2. Content Script (mới, riêng cho auto-reply)

src/content/features/auto-reply/
  index.ts          ← đăng ký onMessage từ SW, gửi kết quả về
  dom.ts            ← SELECTORS (chat list row, message bubble, input, send)
  scrape.ts         ← findUnread(), readLastMessage(convId), openConv(convId)
  act.ts            ← typeAndSend(text) — gõ vào input rồi click send
  wait.ts           ← waitForSelector, waitForText, retry helper

Đăng ký qua manifest.config.ts với matches: ['https://www.zhipin.com/web/geek/chat*']

3. Popup (mở rộng popup hiện có)

src/popup/
  App.tsx           ← +toggle ON/OFF, +counter, +trạng thái scheduler
  ConfigForm.tsx    ← (mới) — ô nhập API key, chỉnh throttle/limit, lịch

4. Shared (mới)

src/shared/
  messages.ts       ← type Msg =
                    │   { type: 'SCRAPE_UNREAD' }
                    │   { type: 'REPLY', convId, text }
                    │   { type: 'TOGGLE', on: bool }
                    │   { type: 'GET_STATE' }
  types.ts          ← Conversation, DailyStats, AppConfig
  prompt.ts         ← HARDCODED_SYSTEM_PROMPT (chuỗi tiếng Việt/Anh)

5. Manifest thay đổi

manifest.config.ts
  + host_permissions: thêm 'https://www.zhipin.com/*'
  + content_scripts: matches đổi từ 'https://*/*' sang
                     'https://www.zhipin.com/web/geek/chat*'
                     (vẫn giữ content/features/index.ts cho oocl.com stub
                      bằng cách tách thành 1 entry khác)
  + permissions: thêm 'alarms'

Data flow (một reply)

  1. SW: alarm fires → loop.runOnce()
  2. SW checks config: enabled? in schedule window? daily < limit? → nếu fail, return.
  3. SW gọi chrome.tabs.query({ url: 'https://www.zhipin.com/web/geek/chat*' }) → lấy tabId.
  4. SW gửi { type: 'SCRAPE_UNREAD' } → content script.
  5. Content script: scrape.findUnread() → trả về Conversation[] (chưa lọc dedupe).
  6. SW lọc qua repliedSet trong storage → lấy conv đầu tiên chưa reply.
  7. SW: scrape.openConv(convId) → đợi DOM render → readLastMessage().
  8. SW: gemini.generateReply(systemPrompt, history) → reply text.
  9. SW: act.typeAndSend(text) → content script gõ + click send.
  10. SW: cập nhật repliedSet, dailyStats.sent++, lastReplyAt.
  11. SW: setTimeout(loop.runOnce, jitter(2000, 5000)) cho cuộc tiếp theo.

Nếu tách file như trên ổn, tôi sang phần State / storage schema.
